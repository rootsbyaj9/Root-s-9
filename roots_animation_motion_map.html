The animation philosophy for Root's
One rule above all: Every animation serves the client's emotion — desire, trust, or urgency. If it doesn't make the visitor feel something or understand something faster, it doesn't belong. Premium feel comes from restraint + precision, not from quantity. The goal is for the site to feel like it's breathing — alive but unhurried.
Performance contract: Every animation listed uses only transform and opacity — the two CSS properties the browser composites on the GPU. Zero layout recalculation. Zero paint. 60fps on every device including mid-range Android phones in Hyderabad.

Global animations — applied across every page
Lenis smooth scroll — the single biggest upgrade
Install Lenis (npm install lenis). One import in _app.tsx. Immediately makes every page feel premium. Replaces the jarring default browser scroll with fluid, physics-based motion. Every Awwwards winner uses this or similar.
Page transitions — ivory curtain between all navigations
Framer Motion AnimatePresence in layout.tsx. An ivory (#FEFCF8) overlay slides down over the page on exit, then slides up to reveal the new page. Duration 400ms per direction. Total transition: 0.8s. Feels like turning a magazine page.
Sticky "Book Now" CTA — appears after hero scrolls out
After the hero section scrolls fully past, a floating orange pill slides up from the bottom of the screen: translateY(80px → 0). Stays visible throughout the entire session. On mobile this is the primary conversion trigger. Uses position: fixed bottom-right, above WhatsApp button.
Footer entrance — horizontal rule draw
When footer enters viewport: the orange divider line at the top of the footer grows from left to right using scaleX(0→1), transform-origin left. Duration 0.8s. Then ROOT'S logo fades in, then columns stagger up. The footer feels like a final reveal, not an afterthought.
WhatsApp button — always present, never annoying
Fades in 2s after page load. On hover: translateY(-4px) + stronger green shadow. On mobile: slightly larger (56px vs 48px) for thumb access. Never animated beyond hover — persistent elements should not demand attention.
Reduced motion — accessibility non-negotiable
All animations must be wrapped in @media (prefers-reduced-motion: no-preference). Users with vestibular disorders, epilepsy, or older devices get instant transitions. Framer Motion: use useReducedMotion() hook. GSAP: check window.matchMedia.
